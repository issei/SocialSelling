1 Visão & escopo

O produto torna a prospecção eficiente e automática: dado um ICP (Ideal Customer Profile), o sistema descobre candidatos, infere dados estruturados, pontua o fit/intent/confiança e entrega Lead Cards ranqueados com justificativa (XAI).

No escopo

  • Ranking explicável de prospects (fit · intent · confiança)
  • Descoberta + enriquecimento incremental de leads
  • Justificativa estruturada (drivers +/− e sinais ausentes)

Fora do escopo (guardrails)

  • CRM, automação de outreach/mensageria, cadências
  • Banco de dados, Redis, Celery, Docker, AWS/Terraform
  • Scraping de Instagram/LinkedIn

2 Arquitetura do PoC

Um único processo Python 3.11+, sem servidor obrigatório, sem banco, sem fila. Persistência em JSON atômico (write-temp + os.replace). O único custo variável são os tokens de duas APIs de cognição/busca.

3 Camadas semânticas & regras invioláveis

A correção do sistema repousa sobre quatro invariantes. Elas valem para todo código novo — inclusive os sensores e a camada de volume.

Observed Evidence
camada 1 · o que foi visto
≠
Inferences
camada 2 · o que foi deduzido
≠
Evaluated Hypotheses
camada 3 · score & XAI

① Isolamento de camadas

Uma inferência jamais é tratada como evidência. Sem referências mutáveis compartilhadas. Apollo vira ObservedEvidence, nunca inferência.

② Determinismo byte-idêntico

Mesma memória → mesma saída. Relógio e RNG injetados; IDs por hash estável; tie-break estável.

③ Open-World

Ausência de sinal = incerteza (↑u), nunca falso. Missing Evidence é explícito; falhas degradam, não quebram.

④ Persistência atômica

Toda escrita é write-temp + os.replace. Nunca um arquivo meio-escrito — vale para cache, corpus e ledgers.

4 Decisões arquiteturais (ADRs)

Cada decisão consciente vira um ADR versionado e referenciável. A fonte da verdade é o ADR-000; os demais são emendas explícitas.

ADR-000canônico

Escopo canônico do PoC local

Fixa SDD v1.0 como espinha; database-less; Tavily+Gemini; M1–M5 como pipeline.
ADR-001

Camada de intenção: hipóteses

Intent = Σ priors das hipóteses ativadas; desqualificadores zeram o lead (hard filter).
ADR-002

UI de operador local (FastAPI)

Superfície opcional para ver/editar params, assistente Gemini e Lead Cards.
ADR-003diferido

Orquestração paralela (LangGraph) + FinOps

Motor async opcional para colheita multi-provedor e poda precoce. Diferido por design.
ADR-004implementado

Apollo.io como 2º sensor firmográfico

Escada de enriquecimento incremental: descoberta grátis → org-enrich → reveal por crédito.
ADR-005core

Reforma cognitiva: lotes + orçamento RPD

Extração em lotes eleva o teto Gemini; orçamento de requisições/dia + ondas resumíveis.
ADR-006core

Corpus de leads acumulativo

Runs deixam de sobrescrever e passam a acumular (upsert idempotente por entity_id). É a maior alavanca de volume: o esforço de cada dia persiste.

6 Boas práticas de desenvolvimento

É aqui que o projeto pretende ser modelo: a complexidade vive no processo de construção, não no sistema. O PoC permanece enxuto; a disciplina é total.

SDD-to-Code Loop (obrigatório por módulo)

  1. 1Contrato Pydantic
  2. 2BDD + fixtures
  3. 3Implementação mínima
  4. 4Gate verde
  5. 5PR + auto-merge

Quality Gate inegociável

ruff + mypy --strict + pytest 100% verde e determinístico. Flakiness = falha.

Git: branch → PR → auto-merge

Nunca commit direto na main. CI obrigatório; merge só com gate verde. Squash + delete-branch.

Tags = pontos de restauração

Cada WU verde recebe uma tag anotada. Rollback público via git revert, nunca reset/force.

APIs sempre mockadas

Testes nunca tocam rede: fixtures gravadas + fakes. Tolerância numérica 1e-9.

Opt-in = paridade

Toda feature nova entra desligada por padrão → o baseline permanece byte-idêntico (smoke E2E).

FinOps por ledger atômico

Orçamentos de crédito (mensal) e de requisições (diário) persistidos; try_spend/refund/reconciliação.

Comando de gate (Windows)

.venv\Scripts\python.exe -m ruff check .   # lint
.venv\Scripts\python.exe -m mypy           # tipos (strict)
.venv\Scripts\python.exe -m pytest -q       # 120 testes determinísticos

7 Estratégia de volume (rodando local)

Objetivo: buscar muito mais leads sem sair do local/database-less. Aplicou-se Teoria das Restrições ao funil — ampliar descoberta sem elevar o teto cognitivo só faz bater no muro mais rápido. Daí os quatro pilares.

PilarO que fazEstado
A · Teto cognitivoExtração em lotes + orçamento RPD + ondas resumíveis (ADR-005)✅ core
B · Corpus acumulativoRuns acumulam; upsert idempotente; ranking sobre o corpus inteiro (ADR-006)✅ core
Apollo · DescobertaPeople Search grátis + org-enrich + reveal por crédito (ADR-004)✅ completo
C · Estado escalávelJSON monolítico → NDJSON/shards (sem banco)⏸ diferido
D · Entity resolutionDomínio canônico + exclusão de vendors + dedup cross-provider⏸ diferido

Tudo opt-in ([apollo].enabled · [corpus].enabled · [gemini].rpd_enabled). Diferidos com razão documentada — não há refino arriscado por completude.

8 Processo de auto-aprendizagem

Ao final de cada tarefa, os aprendizados são destilados em docs/licoes-aprendidas.md (formato L-NNN | Categoria | Lição | Como aplicar) e em memória persistente. Hoje são 45 lições que retroalimentam decisões futuras — o projeto fica mais inteligente a cada ciclo.

L-012 · Determinismo

Injetar relógio e derivar IDs por hash estável — nunca UUID aleatório nem datetime.now() interno. Garante reexecução byte-idêntica.

L-026 · Async resiliente

Fan-out com asyncio.gather(return_exceptions=True) + tratar exceção crua: um provedor não derruba o lote.

L-039 · Orquestração de agentes

Agentes em background travam em prompt de permissão → padrão harvest: agentes escrevem, o main loop valida e mergeia.

L-042 · Paridade por opt-in

Feature nova com default desligado mantém o smoke E2E byte-idêntico — evoluir sem regressão.

L-027 · Poda conservadora

Heurística barata prefere falso-prosseguir a falso-podar: perder um cliente custa mais que 1 chamada Gemini.

L-045 · Escopo honesto

“Terminar todas as specs” ≠ implementar cada sub-feature. Diferir refino arriscado com razão documentada é entrega honesta.
